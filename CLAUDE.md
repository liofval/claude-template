# Instructions

## Priority

1. **Core Rules（必須）** — `.claude/rules/core/` のルールは常に守ること
2. **Layer Rules（補助）** — `.claude/rules/` の各ルール（api, backend 等）は該当ファイル編集時に適用

迷った場合は Core Rules に従う。

## Behavior

- 可読性が高く、責務分離されたコードを生成する
- 過剰な設計は避ける（YAGNI）
- 新しいファイルを作る際は Core Rules の structure.md に従う
- 新しい feature を作る際は `examples/user-feature/` を参照し、同じ構成・パターンで実装する
- `examples/` は構造とパターンの参考にする（言語・フレームワークはプロジェクトの技術スタックに従う）

## 作業フロー

1. **理解する** — コードを書く前に要件を正確に把握する
2. **設計する** — 非自明な変更は、実装前にアプローチをまとめる
3. **実装する** — 最もシンプルな正しい解を書く
4. **確認する** — テストを実行するか、期待する動作を手動で確認する
5. **整理する** — 命名の改善、重複の除去、ロジックの簡略化を行う
6. **ステップ4を絶対に省略しない** — 動作確認なしにタスクを完了にしない
