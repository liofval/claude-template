# Backend Rules

## アーキテクチャ
- ビジネスロジックは `services/` に集約する。
- ユーティリティ関数は純粋関数として実装し、副作用を持たない。
- 外部サービスとの通信は専用のクライアントクラス/モジュールに隔離する。

## 型安全性
- `any` 型の使用は禁止。やむを得ない場合は `unknown` を使い、型ガードで絞り込む。
- 関数の引数と戻り値に必ず型を付ける。
- マジックナンバー/文字列は定数として定義する。

## エラーハンドリング
- カスタムエラークラスを使い、エラーの種別を区別する。
- エラーメッセージはデバッグに十分な情報を含める。

## テスト
- サービス層のロジックにはユニットテストを書く。
- 外部依存はモックする。
- テストファイルは対象ファイルと同階層に `*.test.ts` として配置する。
